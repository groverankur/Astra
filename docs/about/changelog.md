# Changelog

## 0.5.1 - 2026-04-08

### Astra branding and docs refresh

- public platform naming updated to Astra while preserving technical `astraauth-*` compatibility
- `astra` documented as the preferred CLI executable
- MKDocs content rewritten to match the shipped packages and runtime flows
- reserved future modules and planned SDKs documented as future additions, not current packages
- minimum supported Python version lowered to `3.12` with UUIDv7 compatibility helpers
- release-facing repository policy files added and release smoke checks aligned to `astra`
